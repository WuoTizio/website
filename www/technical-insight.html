BASIC - "Server and Client"

Description A Jolie program defines a service and is a composition of two parts, called behaviour and deployment. A behaviour defines the implementation of the functionalities offered by a service; behavioural primitives include communication and computation constructs. However, these do not deal with how communications are supported: they abstractly refer to communication ports, which are assumed to be correctly defined in the deployment part.

Goal A simple Server - Client application: the client sends a number (5) to the server. The server return the number received multiplied by two, which will be printed by the client.

Show Tips
Server

main
{
     twice( number )( response ) {
          response = number * 2;
          println@Console( number )
     }
}

Output:

5

Client

main
{
     twice@TwiceService( 5 )( response );
     println@Console( response )
}

Output:

10
